CASE STUDY: Williston Park Library

Building Inclusive Digital Access

Background

This project focused on improving accessibility across the Home, Events, and Contact pages of the Williston Park Public Library website. As part of my senior capstone, I redesigned these pages to meet WCAG 2.2 AA standards and ensure a user-friendly experience for all users, including those who rely on screen readers or keyboard-only navigation. The redesign targeted structure, semantics, color contrast, and overall digital inclusivity.

Challenge

The original site contained several accessibility issues, including missing alt text, unlabeled form inputs, low-contrast color palettes, incorrect ARIA roles, and non-semantic markup. These barriers made navigation difficult for blind and low-vision users and did not meet compliance requirements.

Process

I began with an accessibility audit using Lighthouse, WAVE, and manual keyboard and screen-reader testing. This revealed inconsistent heading structure, unlabeled elements, and incorrect ARIA attributes across all pages. I then rebuilt each interface to improve semantic clarity, operability, and meaningful, perceivable feedback.

  • Added semantic landmarks and a logical heading hierarchy.
  • Wrote descriptive alt text and clarified link labels.
  • Improved color contrast and added visible focus indicators.
  • Rebuilt the calendar with proper table headers and added a list-view alternative.
  • Corrected ARIA roles and ensured every form label was properly associated.
  • Replaced the embedded map with an accessible static image + text alternative.

Results

All pages reached full WCAG 2.2 AA compliance. Lighthouse Accessibility scores improved from 75 to 100 across Home, Events, and Contact pages. Keyboard-only navigation now functions reliably, users receive meaningful feedback, and the entire interface supports assistive technology. The redesign also establishes a repeatable system for maintaining accessibility moving forward.
