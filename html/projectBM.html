CASE STUDY: Birthday Messaging

Designing for Accessible Connection

Background

This project focused on enhancing accessibility within BirthdayMessaging.io, an app that allows users to create and send personalized birthday messages. The goal was to align the messaging flow with WCAG 2.2 AA while maintaining a fun, user-friendly experience.

Key areas included semantic HTML, ARIA for perceivable updates, consistent focus states, and a scalable Tailwind setup.

Challenge

The original experience had accessibility barriers, including low-contrast colors, missing labels, and non-semantic markup, which limited usability for keyboard and screen reader users.

Process

I conducted an accessibility audit using Lighthouse and manual keyboard testing to identify issues. Key improvements included:

  • Structure & semantics: Implemented semantic HTML and added descriptive labels and targeted ARIA attributes.
  • Perceivable feedback: Added ARIA live regions for real-time confirmations and status updates.
  • Keyboard operability: Improved tab order and established consistent, visible focus indicators.
  • Color & contrast: Updated color tokens to meet WCAG 2.2 AA contrast requirements.
  • Systemization: Replaced inline styles with global Tailwind tokens for consistency and maintainability.
  • Form validation: Used aria-describedby to surface clear, associated error messages.
  • Documentation: Created a README and front-end style guide with accessibility notes and code examples.

After each fix, I reran audits in an iterative testing cycle until full compliance was achieved.

Results

Significant outcomes included:

  • A fully keyboard-navigable messaging flow with visible focus indicators.
  • Status and confirmation messages announced via ARIA live regions.
  • All interactive states meeting WCAG 2.2 AA contrast requirements.
  • Inline styles replaced with Tailwind tokens for easier maintenance.
  • Improved Lighthouse scores and stronger manual accessibility test results across pages.
